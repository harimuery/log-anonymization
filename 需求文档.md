## 1. 引言

### 1.1 背景
支付系统在处理交易过程中，日志中会不可避免的包含银行卡号、身份证号、手机号、密码、令牌等敏感信息。为满足 PCI DSS、个人信息保护法等合规要求，同时保障线上问题高效排查能力，需要建设一套全覆盖、自动化、高性能的日志脱敏系统。

### 1.2 目标
基于 Java 生态，提供从日志产生、输出、采集到归档的全生命周期敏感信息识别与脱敏能力，确保敏感数据不被明文记录，且脱敏行为可审计、可追溯、不影响业务连续性。

### 1.3 适用范围
- 所有 Java 技术栈实现的支付核心服务、网关服务、管理后台等。
- 日志框架范围：Logback、Log4j2、SLF4J 等主流框架。
- 日志输出目标：本地文件、ELK、远程采集器等。

---

## 2. 总体描述

### 2.1 用户角色
- **开发者**：在代码中输出日志，依赖脱敏 SDK/插件自动处理。
- **安全管理员**：配置脱敏规则、审计脱敏日志、响应告警。
- **运维人员**：监控脱敏组件性能，处理降级与熔断告警。

### 2.2 系统上下文
系统以 Java Library 形式嵌入业务服务，同时提供集中配置控制台（可选）和日志采集端处理组件。与密钥管理服务（KMS）、令牌化服务、统一日志平台等有集成关系。

## 3. 功能需求

### 3.1 敏感信息自动识别

#### 3.1.1 多级敏感数据类型定义
系统应内置并可扩展以下敏感数据分类：

- **金融数据**：卡号、有效期、CVV、交易密码、支付令牌、账户余额、交易金额（可配置保留）。
- **个人身份信息**：姓名、身份证号、手机号、邮箱、地址、IP、设备指纹。
- **认证凭证**：登录密码、支付密码、短信验证码、Session Token、API Key。
- **系统安全数据**：数据库连接串、加密密钥、内部服务调用凭证。
- **业务扩展类型**：支持通过配置中心动态添加订单号、商户号等自定义类型。

#### 3.1.2 多模式识别引擎
- **正则/规则匹配**：内置并支持自定义正则表达式库（卡号需结合 Luhn 校验，身份证号含校验位）。
- **字段名匹配**：自动识别 `password`、`pan`、`cvv`、`phone` 等 JSON/XML 字段名或 Log Marker 名。
- **上下文关键字匹配**：扫描日志消息文本中形如 `cardNo=xxx`、`password:xxx` 的模式。
- **指纹相似度**：对长数字串或格式化字符串进行特征检测，提示未知潜在敏感数据。
- **白名单机制**：排除流水号、时间戳、UUID 等无需脱敏的字段，防止误杀。

#### 3.1.3 Java 技术实现约束
- 识别引擎应基于可插拔的 SPI（Service Provider Interface）设计。
- 正则引擎推荐使用 `RE2J` 或预编译 `java.util.regex.Pattern` 缓存，确保线性时间复杂度。

---

### 3.2 脱敏策略与算法库

#### 3.2.1 遮蔽/掩码
- 支持固定字符串替换为 `***`。
- 部分遮盖：可配置首/尾保留位数，中间用自定义字符填充（如 `*`、`#`）。
- 示例：手机号 `138****1234`，卡号 `6222 **** **** 0123`。

#### 3.2.2 加密与哈希
- **不可逆加盐哈希**：对密码等应使用 `SHA-256` + 盐，盐值从 KMS 获取并定期轮换。**严格禁止记录 CVV 明文，即便是脱敏后也不得留存，发现即应丢弃或直接哈希。**
- **可逆加密**：对需还原的卡号、身份证等，采用 `AES-256-GCM` 算法，密钥托管于 KMS，解密需独立授权与审计。
- **保留格式加密**：提供算法选项，加密后密文长度与格式与原值一致（如卡号加密后仍为 16 位数字），便于数据库兼容。
- **令牌化**：将敏感值替换为 Token，通过内部令牌化服务可兑换原始值，支持跨系统关联。

#### 3.2.3 聚合与泛化
- 金额可配置取整为区间（如 `>5000`）。
- IP 可掩码为 `10.1.*.*` 或仅保留前两段。
- 地理位置泛化到省/市级别，日期泛化为年月。

#### 3.2.4 随机化与仿真
- 支持生成符合格式规则的虚假数据（如通过姓名库生成随机姓名，通过 Luhn 算法生成假卡号），用于测试环境日志，确保不泄露真实信息且不破坏日志格式有效性。

#### 3.2.5 Java 技术实现约束
- 所有脱敏算法抽象为统一接口 `SensitiveDataMasker`，策略通过工厂模式或注解选择。
- 加密算法调用 Java Cryptography Architecture (JCA)，集成 KMS SDK。

---

### 3.3 脱敏规则配置管理

#### 3.3.1 分层级作用域
支持优先级由低到高的规则覆盖：
- **全局默认**：适用于所有日志。
- **应用级别**：针对特定服务。
- **日志 Appender/输出器级别**：控制台、文件分别配置。
- **包/类级别**：通过包名或类名匹配。
- **特定日志事件级别**：可基于 `Marker`、`Level` 指定规则。

#### 3.3.2 可视化策略编排
- 提供管理控制台界面（或与配置中心集成），支持按“字段路径 + 敏感类型 + 脱敏算法”组合规则，无需修改代码。
- 规则版本管理、历史追溯、灰度发布及一键回滚。

#### 3.3.3 条件与例外处理
- 按环境（dev/test/prod）自动切换规则集。
- 支持审批流程下的临时豁免：经审批后，可在限定时间、限定范围内输出部分字段明文，所有操作全程审计。
- 支持黑白名单叠加：如针对特定用户 ID 的日志可展示手机号明文（需严格管控）。

#### 3.3.4 敏感类型与算法的绑定模板
- 内置预定义模板：身份证→部分遮盖，卡号→保留前6后4并加密存储原值，密码→哈希。
- 支持一键应用模板，防止遗漏。

#### 3.3.5 Java 技术实现约束
- 配置可通过 YAML/XML 文件加载，同时适配 Spring Cloud Config、Nacos 等配置中心动态刷新。
- 规则数据结构使用 `List<Rule>`，支持链式匹配，高性能优先。

---

### 3.4 脱敏时机与处理管道

#### 3.4.1 日志产生端预脱敏
- 提供 Java 工具类 `SecureLogger`，在业务代码中调用其方法进行预脱敏后打印，适用于对性能极致敏感场景。

#### 3.4.2 日志框架层面插桩（推荐方式）
- **Logback 实现**：自定义 `MessageConverter` 或 `PatternLayout`，拦截日志消息、参数、异常堆栈，在输出前执行脱敏。
- **Log4j2 实现**：使用 `RewriteAppender` 或自定义 `LogEventPatternConverter` 进行字段扫描。
- 支持对 MDC 中的值、异常链中的 `message` 和 `toString()` 进行全面扫描。
- 支持对 SLF4J 参数化占位符传入的对象进行递归字段脱敏，防止敏感对象 `toString()` 泄露。

#### 3.4.3 日志收集端脱敏（集中式）
- 在 Filebeat、Logstash 等管道中，提供 Java 自定义插件或通过调用脱敏服务 API 对已落盘的结构化日志进行二次清洗，作为最后防线。
- 支持对非结构化文本的二次识别与处理。

#### 3.4.4 异步与离线脱敏
- 提供离线批处理工具（基于 Spring Batch 或独立 Java 应用），对历史归档日志文件进行脱敏扫描与重写，满足数据留存合规要求。

#### 3.4.5 Java 技术实现约束
- 框架插桩方案应支持异步日志输出，避免阻塞业务线程。
- 脱敏上下文在跨线程传递（如 `MDC` 透传）时确保规则一致。

---

### 3.5 审计与合规支撑

#### 3.5.1 脱敏操作审计日志
- 记录每条规则命中次数、脱敏成功/失败数量、敏感类型分布统计。
- 规则变更、审批豁免、密钥访问、解密操作均需记录不可篡改的审计日志，并同步至安全信息与事件管理（SIEM）系统。
- 审计日志本身不含敏感明文。

#### 3.5.2 合规报告能力
- 按 PCI DSS、GDPR、个人信息保护法要求，生成脱敏策略合规映射报告。
- 证明 CVV、磁道数据等禁止存储的数据未被记录，密码使用加盐哈希且不可还原。
- 展示数据保留周期：已脱敏数据可保留，到达期限的明文自动删除。

#### 3.5.3 敏感数据存留扫描
- 定期启动后台任务扫描所有日志存储（包括冷备），发现意外明文（如卡号）时生成告警并自动触发补救脱敏。

#### 3.5.4 Java 技术实现约束
- 审计日志通过独立 Appender 写入审计专用日志文件，不得与业务日志混用。
- 扫描任务可利用 Spring Scheduler 或 Quartz 实现，配合分布式执行框架。

---

### 3.6 性能与稳定性保障

#### 3.6.1 高性能模式匹配
- 规则预编译，敏感词匹配采用多模式匹配算法（Aho-Corasick）优先，降低正则开销。
- 对超大日志消息（如超过 4KB）支持截断处理或跳过扫描，避免 GC 抖动。
- 支持采样脱敏：对高并发交易日志可配置按百分比脱敏。

#### 3.6.2 异步与非阻塞处理
- 日志脱敏写入必须完全异步，利用 `AsyncAppender` 或 `Disruptor` 等高性能队列，不阻塞业务线程。
- 可逆加密调用 KMS 时，必须使用本地缓存 + 异步刷新，禁止每条日志同步远程调用。

#### 3.6.3 熔断与降级
- 当脱敏组件自身耗时超过阈值（如 10ms）或故障率上升时，自动降级为简单全局遮盖并写入标记，同时触发严重告警。
- 降级期间，所有日志以“脱敏降级”占位符输出，保证业务日志不丢失。
- 提供独立的线程池和资源隔离，防止脱敏逻辑耗尽业务资源。

#### 3.6.4 Java 技术实现约束
- 可集成 Resilience4j 实现熔断、限流。
- 利用 `Micrometer` 暴露性能指标，接入监控系统。

---

### 3.7 测试与开发支持

#### 3.7.1 脱敏有效性测试框架
- 提供 JUnit 5 扩展或断言工具，允许在单元测试中验证日志输出不包含明文敏感信息。
- 内置敏感数据集（测试卡号、假身份证号），方便编写验证用例。

#### 3.7.2 开发环境模拟
- 开发/测试环境默认开启“强校验模式”：一旦检测到可能泄露敏感信息的日志调用，立即在控制台高亮告警或抛出异常，强制开发者使用脱敏方式。
- 提供 IDE 插件（可选）：识别 `log.info("cardNo:" + cardNo)` 等危险用法并标记。

#### 3.7.3 Java 技术实现约束
- 测试依赖以 JAR 包形式提供，内嵌脱敏规则检测器。
- 支持通过 Spring Profile 切换校验模式。

---

### 3.8 异常处理与告警

#### 3.8.1 脱敏失败处理策略
- 可配置失败行为：
  - 屏蔽整个字段/消息（替换为“脱敏失败”）。
  - 使用失败占位符并写入原始值加密后的密文（需确保密文安全）。
  - **严禁直接输出原始明文**。
- 记录失败计数器并上报告警。

#### 3.8.2 攻击/探测告警
- 短时间内同一节点触发大量敏感规则命中（如扫描日志注入），视为攻击行为，触发安全告警。
- 检测到明文 CVV、磁道数据等根本不应存在的数据类型时，立即发送最高级别告警，并通知安全团队。
- 告警信息通过邮件、企业微信、钉钉等渠道通知，并可集成至 SIEM。

#### 3.8.3 Java 技术实现约束
- 告警逻辑应异步发送，避免干扰主流程。
- 异常处理需确保日志框架本身不会因脱敏异常而崩溃。

---

### 3.9 与其他系统的集成

#### 3.9.1 KMS 集成
- 支持对接阿里云 KMS、AWS KMS、自建 Vault 等，实现密钥生命周期管理。
- 加密密钥与脱敏组件解耦，支持密钥轮换无感知。

#### 3.9.2 令牌化服务集成
- 如企业已有统一令牌化服务，脱敏组件可通过 SDK/HTTP 调用将明文替换为 Token，并支持批量兑换。

#### 3.9.3 日志平台与 SIEM 集成
- 脱敏后的日志保持结构化（JSON），字段含义不变，便于 ELK 等平台解析。
- 提供审计日志接口，供 SIEM 采集。

## 4. 非功能需求

### 4.1 性能要求
- 对单条日志的脱敏处理平均延迟 < 1ms，P99 延迟 < 5ms。
- 在开启全面脱敏的情况下，TPS 不低于原系统的 95%。

### 4.2 安全性要求
- 脱敏规则配置文件中的密钥引用不直接暴露明文，使用占位符并运行时解密。
- 所有加解密逻辑必须通过安全评审，禁止自行设计算法。
- 降级场景下绝不能输出原始明文。

### 4.3 可维护性
- 规则增加、调整无需重启服务（动态刷新）。
- 提供 Java API 用于自排查，可输出当前生效规则和脱敏效果预览。

### 4.4 可扩展性
- 支持自定义识别器和脱敏算法插件，遵循 SPI 机制。
- 支持多应用接入时，SDK 轻量且无外部依赖冲突。

## 5. 技术约束

- **运行环境**：JDK 11+，兼容 JDK 17 LTS。
- **日志框架**：必须同时支持 Logback 1.2+ 和 Log4j2 2.x，可选支持 SLF4J 绑定。
- **应用框架**：兼容 Spring Boot 2.5+ 及 Spring Boot 3.x。
- **构建工具**：Maven / Gradle，提供 BOM 进行版本管理。
- **加密库**：集成 Java 内置 JCA，推荐 Bouncy Castle 作为补充。
- **测试**：JUnit 5 + AssertJ。

## 6. 术语表

| 术语         | 描述                                     |
| ------------ | ---------------------------------------- |
| 脱敏         | 将敏感信息替换为不可识别的替代字符的过程 |
| 遮蔽         | 通过部分隐藏、替换等方式让数据失去识别性 |
| 令牌化       | 以无意义的 Token 替代真实数据            |
| 保留格式加密 | 加密后数据保持与原始数据相同长度和格式   |
| 熔断         | 在系统部分故障时自动切换为安全模式的能力 |
