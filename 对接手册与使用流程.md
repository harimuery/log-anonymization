## 一、快速开始（5 分钟接入）

### 1.1 Maven 依赖

在业务应用的 `pom.xml` 中添加 starter 依赖：

```xml
<dependency>
    <groupId>com.example.anonymization</groupId>
    <artifactId>log-anonymization-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

**依赖传递关系**：

```
log-anonymization-spring-boot-starter
├── log-anonymization-core          (核心脱敏引擎)
├── log-anonymization-api           (领域模型与端口)
├── log-anonymization-logback       (Logback 集成，可选)
├── log-anonymization-log4j2        (Log4j2 集成，可选)
├── resilience4j-circuitbreaker     (熔断器)
├── caffeine                        (本地缓存)
├── disruptor                       (高性能审计队列)
└── flyway-core                     (数据库迁移，optional)
```

### 1.2 最小配置

在 `application.yml` 中启用脱敏（默认已启用）：

```yaml
log-anonymization:
  enabled: true
  rule-file-path: classpath:log-anonymization-rules.yml
```

### 1.3 验证接入

编写一个简单的 Spring Boot Test：

```java
@SpringBootTest
class LogAnonymizationIntegrationTest {

    @Test
    void shouldMaskBankCard() {
        String message = "用户银行卡号：6222021234567890123，支付成功";
        String masked = SecureLogger.mask(message);
        
        // 验证：银行卡号中间部分被遮盖
        assertThat(masked).contains("622202*********0123");
        assertThat(masked).doesNotContain("123456789");
    }
}
```

**首次运行输出示例**：

```
2026-06-22 10:30:15.123  INFO  [main] c.e.a.c.a.SecureLogger : 
    脱敏完成 | 原始长度: 32 | 处理后长度: 32 | 动作: MASKED | 数据类型: BANK_CARD
```

## 二、配置详解

### 2.1 完整配置模板

```yaml
log-anonymization:
  # 总开关
  enabled: true
  
  # 规则来源：LOCAL_FILE（默认）/ NACOS
  rule-source: LOCAL_FILE
  rule-file-path: classpath:log-anonymization-rules.yml
  
  # Nacos 配置（rule-source=NACOS 时生效）
  nacos:
    server-addr: localhost:8848
    data-id: log-anonymization-rules
    group: DEFAULT_GROUP
  
  # 密钥配置
  secret:
    salt: ${MASKING_SALT:default-salt-change-in-production}
    jasypt-enabled: false
  
  # 审计配置
  audit:
    enabled: true
    batch-size: 100
    ring-buffer-size: 65536
    flush-interval-seconds: 5
  
  # 采样配置（压测场景可开启）
  sampling:
    enabled: false
    rate: 1.0
  
  # 验证配置
  validation:
    fail-fast: true
  
  # === 线程池配置（P3 新增）===
  thread-pool:
    audit:
      core-pool-size: 1
      max-pool-size: 2
      queue-capacity: 5000
      keep-alive-seconds: 60
      daemon: true
  
  # === 缓存配置（P3 新增）===
  cache:
    caffeine:
      rule-cache:
        max-size: 1000
        expire-after-write-seconds: 300
        record-stats: true
      pattern-cache:
        max-size: 500
        expire-after-access-seconds: 1800
        record-stats: true
  
  # === 熔断器配置（P3 新增）===
  circuit-breaker:
    enabled: true
    failure-rate-threshold: 50
    slow-call-rate-threshold: 100
    slow-call-duration-threshold: 10ms
    sliding-window-size: 100
    sliding-window-type: COUNT_BASED
    wait-duration-in-open-state: 30s
    permitted-number-of-calls-in-half-open-state: 10
    automatic-transition: true
  
  # === Flyway 数据库迁移（P3 新增）===
  flyway:
    enabled: false
    locations: classpath:db/migration
    baseline-on-migrate: true
    table: flyway_schema_history_anonymization
  
  # === 灰度发布配置（P3 新增）===
  gray-release:
    percent: 0
    old-version: "1.0.0"
    new-version: "1.0.0"
```

### 2.2 配置项说明表

| 配置项 | 默认值 | 说明 | 调优建议 |
|--------|--------|------|----------|
| `enabled` | `true` | 总开关 | 压测时设为 `false` 完全关闭 |
| `rule-source` | `LOCAL_FILE` | 规则来源 | 生产环境建议用 `NACOS` |
| `thread-pool.audit.*` | 见模板 | 审计线程池 | 高并发场景调大 `queue-capacity` |
| `cache.caffeine.*` | 见模板 | Caffeine 缓存 | 规则频繁变更时减小 `expire-after-write` |
| `circuit-breaker.*` | 见模板 | Resilience4j 熔断 | 延迟敏感场景调低 `slow-call-duration-threshold` |
| `flyway.enabled` | `false` | Flyway 迁移 | 首次部署设为 `true`，之后可关闭 |
| `gray-release.percent` | `0` | 灰度比例 | 升级时逐步调大：5→10→25→50→100 |

## 三、编程式 API 使用

### 3.1 SecureLogger — 核心入口

`SecureLogger` 是面向业务开发者的**外观模式（Facade）**入口，提供 4 类方法：

#### 3.1.1 单字段脱敏（纳秒级）

```java
// 按数据类型脱敏单个值
String bankCard = "6222021234567890123";
String masked = SecureLogger.mask(bankCard, SensitiveDataType.BANK_CARD);
// 输出: "622202*********0123"

String phone = "13800138000";
String maskedPhone = SecureLogger.mask(phone, SensitiveDataType.PHONE);
// 输出: "138****8000"

String idCard = "110101199001011234";
String maskedId = SecureLogger.mask(idCard, SensitiveDataType.ID_CARD);
// 输出: "110***********1234"
```

**性能特征**：
- 单次调用 P99 延迟 < 1μs（微秒）
- 无锁实现，线程安全
- 适用于高频调用场景（如日志输出、接口响应）

#### 3.1.2 整条消息脱敏（完整管道）

```java
// 走完整脱敏管道：BloomFilter → Detection → Masking → Audit
String message = "用户张三，手机号13800138000，银行卡6222021234567890123";
String masked = SecureLogger.mask(message);
// 输出: "用户张三，手机号138****8000，银行卡622202*********0123"
```

**管道阶段说明**：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ BloomFilter │ → │  Detection  │ → │   Masking   │ → │    Audit    │
│   预筛阶段   │    │   检测阶段   │    │   脱敏阶段   │    │   审计阶段   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
   ~50ns/hash        ~1-5ms/条         ~0.5ms/条         异步，零阻塞
```

#### 3.1.3 带上下文的脱敏（高级）

```java
LogContext context = LogContext.builder()
    .message("订单支付成功，卡号6222021234567890123")
    .appName("payment-service")
    .loggerName("com.example.payment.OrderService")
    .environment("prod")
    .mdc(Map.of("traceId", "abc-123", "userId", "10086"))
    .build();

MaskingResult result = SecureLogger.mask(message, context);

// 获取详细结果
System.out.println("是否修改: " + result.isChanged());
System.out.println("脱敏动作: " + result.getAction());  // MASKED / SKIPPED / DEGRADED
System.out.println("是否降级: " + result.isDegraded());
System.out.println("脱敏后: " + result.getMasked());
```

#### 3.1.4 对象安全打印

```java
// 自动识别 @SensitiveField 注解字段
public class User {
    private String name;
    
    @SensitiveField(type = SensitiveDataType.PHONE)
    private String phone;
    
    @SensitiveField(type = SensitiveDataType.BANK_CARD)
    private String bankCard;
}

User user = new User("张三", "13800138000", "6222021234567890123");
String safe = SecureLogger.safeToString(user);
// 输出: User{name='张三', phone='138****8000', bankCard='622202*********0123'}
```

### 3.2 在 Logback 中使用

```xml
<!-- logback-spring.xml -->
<configuration>
    <conversionRule 
        conversionWord="masked" 
        converterClass="com.example.anonymization.logback.MaskingConverter"/>
    
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <!-- 使用 %masked 替代 %msg -->

```

| `银行卡6222021234567890123扣款100元` | `银行卡622202*********0123扣款100元` |

## 四、规则配置

### 4.1 YAML 规则文件格式

```yaml
log-anonymization:
  version: 1  # 规则文件版本（用于版本兼容检查）
  
  default-rules:
    # 规则1: 银行卡号脱敏
    - data-type: BANK_CARD
      priority: 100
      enabled: true
      detector:
        type: REGEX
        patterns: ['\b(?:4[0-9]{12}(?:[0-9]{3})?|5[1-5][0-9]{14}|3[47][0-9]{13}|6(?:011|5[0-9]{2})[0-9]{12})\b']
        enable-luhn-check: true
      masker:
        type: PARTIAL_MASK
        keep-prefix-length: 6
        keep-suffix-length: 4
        mask-char: '*'
    
    # 规则2: 手机号脱敏
    - data-type: PHONE
      priority: 90
      enabled: true
      detector:
        type: REGEX
        patterns: ['\b1[3-9]\d{9}\b']
      masker:
        type: PARTIAL_MASK
        keep-prefix-length: 3
        keep-suffix-length: 4
    
    # 规则3: 密码字段哈希
    - data-type: PASSWORD
      priority: 200
      enabled: true
      detector:
        type: FIELD_NAME
        field-names: [password, pwd, passwd, secret]
      masker:
        type: HASH
        algorithm: SHA-256
    
    # 规则4: CVV 丢弃
    - data-type: CVV
      priority: 250
      enabled: true
      detector:
        type: FIELD_NAME
        field-names: [cvv, cvv2, cvc]
      masker:
        type: DISCARD
    
    # 规则5: 邮箱脱敏
    - data-type: EMAIL
      priority: 80
      enabled: true
      detector:
        type: REGEX
        patterns: ['\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b']
      masker:
        type: PARTIAL_MASK
        keep-prefix-length: 1
        mask-domain: false
    
    # 规则6: IP 地址泛化
    - data-type: IP_ADDRESS
      priority: 70
      enabled: true
      detector:
        type: REGEX
        patterns: ['\b(?:\d{1,3}\.){3}\d{1,3}\b']
      masker:
        type: GENERALIZE
        ip-segments-to-keep: 2
```

### 4.2 规则字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `data-type` | ✅ | 敏感数据类型：`BANK_CARD`/`PHONE`/`ID_CARD`/`EMAIL`/`PASSWORD`/`CVV`/`IP_ADDRESS` |
| `priority` | ✅ | 优先级（数值越大越优先，范围 1-1000） |
| `enabled` | ❌ | 是否启用（默认 `true`） |
| `detector.type` | ✅ | 检测器类型：`REGEX`/`KEYWORD`/`FIELD_NAME`/`COMPOSITE` |
| `detector.patterns` | 条件 | REGEX 检测器的正则列表 |
| `detector.field-names` | 条件 | FIELD_NAME 检测器的字段名列表 |
| `detector.enable-luhn-check` | ❌ | 是否启用 Luhn 校验（银行卡场景推荐） |
| `masker.type` | ✅ | 脱敏类型：`PARTIAL_MASK`/`FULL_MASK`/`HASH`/`DISCARD`/`GENERALIZE` |
| `masker.keep-prefix-length` | 条件 | PARTIAL_MASK 保留前缀字符数 |
| `masker.keep-suffix-length` | 条件 | PARTIAL_MASK 保留后缀字符数 |
| `masker.mask-char` | ❌ | 掩码字符（默认 `*`） |
| `masker.algorithm` | 条件 | HASH 算法名（如 `SHA-256`） |

## 五、高级特性

### 5.1 自定义检测器（SPI 扩展）

实现 `SensitiveDataDetector` 接口：

```java
@Component
public class CustomApiKeyDetector implements SensitiveDataDetector {
    
    @Override
    public List<DetectionResult> detect(LogContext context) {
        String message = context.getMessage();
        List<DetectionResult> results = new ArrayList<>();
        
        // 自定义检测逻辑：识别 API Key
        Pattern pattern = Pattern.compile("sk-[a-zA-Z0-9]{48}");
        Matcher matcher = pattern.matcher(message);
        while (matcher.find()) {
            results.add(new DetectionResult(
                matcher.start(), 
                matcher.end(), 
                SensitiveDataType.CUSTOM,
                "API_KEY"
            ));
        }
        return results;
    }
    
    @Override
    public SensitiveDataType getSupportedType() {
        return SensitiveDataType.CUSTOM;
    }
    
    @Override
    public String getDetectorName() {
        return "API-KEY-DETECTOR";
    }
}
```

**注册方式**（三选一）：
1. **Spring Bean**：直接加 `@Component`，自动扫描注册
2. **SPI 机制**：在 `META-INF/services/com.example.anonymization.api.spi.SensitiveDataDetector` 中添加实现类全名
3. **手动注册**：`detectorRegistry.register(new CustomApiKeyDetector())`

### 5.2 自定义脱敏器

```java
@Component
public class CustomApiKeyMasker implements SensitiveDataMasker {
    
    @Override
    public MaskingResult mask(String original, MaskerConfig config) {
        // 保留前 8 位，其余用 *** 替代
        if (original.length() <= 8) {
            return MaskingResult.masked(original, "***");
        }
        String masked = original.substring(0, 8) + "***";
        return MaskingResult.masked(original, masked);
    }
    
    @Override
    public MaskerType getMaskerType() {
        return MaskerType.PARTIAL_MASK;
    }
    
    @Override
    public boolean isReversible() {
        return false;
    }
}
```

## 六、版本兼容与灰度发布

### 6.1 语义化版本控制

SDK 采用 **SemVer 2.0.0** 规范：`MAJOR.MINOR.PATCH`

| 版本变化 | 兼容性 | 示例 |
|----------|--------|------|
| `MAJOR` 升级 | ❌ 不兼容 | `1.0.0` → `2.0.0` |
| `MINOR` 升级 | ✅ 兼容 | `1.0.0` → `1.1.0` |
| `PATCH` 升级 | ✅ 兼容 | `1.0.0` → `1.0.1` |

### 6.2 规则文件版本兼容

规则文件头部声明版本：

```yaml
log-anonymization:
  version: 1  # 规则文件版本
```

**兼容性检查逻辑**：
- 规则版本 `MAJOR` = SDK 版本 `MAJOR` → ✅ 兼容加载
- 规则版本 `MAJOR` ≠ SDK 版本 `MAJOR` → ❌ 抛出 `IncompatibleRuleVersionException`
- 规则版本落后 2 个 `MAJOR` → ⚠️ 记录警告日志

### 6.3 灰度发布配置

**场景**：将 SDK 从 `1.0.0` 升级到 `2.0.0`，先灰度 10% 流量验证。

```yaml
log-anonymization:
  gray-release:
    percent: 10           # 灰度比例 10%
    old-version: "1.0.0"  # 旧版本
    new-version: "2.0.0"  # 新版本
```

**灰度路由原理**：

```
┌─────────────────┐
│   请求进入       │
│  traceId=abc-123 │
└────────┬────────┘
         ▼
┌─────────────────┐
│ FNV-1a 哈希计算  │
│ hash("abc-123") │
└────────┬────────┘
         ▼
┌─────────────────┐
│  hash % 100 = 37 │
│  37 < 10?       │
│  No → 走 v1.0.0  │
└─────────────────┘
```

**关键特性**：
- **确定性路由**：相同 `traceId` 永远路由到同一版本
- **均匀分布**：FNV-1a 哈希保证流量均匀分散
- **原子切换**：`AtomicReference` 无锁更新灰度比例
- **安全降级**：`percent=0` 全走旧版本，`percent=100` 全走新版本

**动态调整灰度比例（运行时）**：

```java
@Autowired
private GrayReleaseRouter grayReleaseRouter;

// 逐步放量
grayReleaseRouter.setGrayPercent(10);   // 10% 流量
grayReleaseRouter.setGrayPercent(25);   // 25% 流量
grayReleaseRouter.setGrayPercent(50);   // 50% 流量
grayReleaseRouter.setGrayPercent(100);  // 100% 流量，升级完成
```

## 九、故障排查

### 9.1 常见问题速查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 敏感数据未被脱敏 | 规则未启用 / 优先级冲突 | 检查 `enabled=true`，调整 `priority` |
| 脱敏后性能下降 | 正则过于复杂 / 缓存未命中 | 简化正则，检查缓存命中率指标 |
| 熔断器频繁打开 | 下游依赖超时 / 错误率过高 | 调大 `sliding-window-size`，检查下游服务 |
| 审计日志丢失 | Disruptor 队列溢出 | 调大 `ring-buffer-size`，检查 Exporter |
| 规则热更新不生效 | 文件监听失败 | 检查 `rule-file-path` 路径，查看日志 |
| 灰度路由不一致 | traceId 未传递 | 确保 MDC 中 `traceId` 已设置 |
